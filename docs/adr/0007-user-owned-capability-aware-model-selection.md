# Use user-owned, capability-aware model configurations

Business model calls use the LiteLLM Python SDK directly with an optional User-owned BYOK, rather than a globally configured LiteLLM proxy or deployment-owned endpoint registry. A Model Configuration owns the adapter, optional private OpenAI-compatible endpoint, model identity, declared capabilities, constraints, and optional encrypted credential; read-only Model Configuration Templates only prefill a new configuration. A credential-free private endpoint receives no API key and may authenticate at its own network or service boundary. This keeps each User's private models available only to that User's tasks while avoiding deployment configuration and tenant concepts before they are needed.

## Consequences

- A User can create an OpenAI-compatible private model configuration with or without a credential, declare the stages it supports, and verify each claimed capability where a non-billable probe exists. A credential-free configuration or one without a safe probe remains unverified until its first real invocation.
- A new Video Task copies the selected configuration reference for each stage. The stage resolves and freezes its non-sensitive adapter, endpoint identity, model identity, constraints, configuration revision, and selection version atomically when it starts; changing Stage Model Defaults later never changes an existing task.
- System backoff retries use that frozen invocation snapshot and never silently change model. A User-triggered retry or regeneration resolves the latest revision of the currently selected configuration, writes a new selection version and Model Resolution Snapshot, and creates a new candidate or execution attempt.
- Credentials remain encrypted, are decrypted only at the Model Invocation Boundary, are never recorded in task state or Generation Records, and may be rotated without changing a frozen model identity. Revocation preserves historical provenance and blocks a later resolution.
- Planning, scriptwriting, keyframe image generation, clip video generation, voice generation, and viral analysis/transcription have separate selections. Deterministic final composition remains outside model selection; optional composition text optimization uses creative planning.
